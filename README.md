# ⬛ AI Image Studio // The Secure Terminal

![AI Image Studio](https://aiimagestudio.s3.us-east-2.amazonaws.com/aiimagestudio.webp)

**AI Image Studio** is a high-performance, zero-retention frontend terminal built for generating hyper-realistic, uncapped AI art. 

Built on a lightweight `PHP`, `Vanilla JS`, and `CSS` stack, this application acts as a secure gateway to the **Pollinations.ai** compute grid. By utilizing a "Bring Your Own Pollen" (BYOP) OAuth flow, the system guarantees sovereign key access—meaning you own your compute, and your prompts bypass standard corporate safety filters.

### ⚡ The Architecture
- **Model:** Hard-locked to the `nanobanana` model for maximum fidelity and uncensored output.
- **Fail-Open Routing:** Bypasses CORS preflight checks by directly injecting authorization tokens into secure `GET` parameters.
- **Ghost Router:** A custom lightweight JS router that seamlessly transitions unauthorized traffic to the VSL landing page, and authorized traffic to the engine dashboard without page reloads.
- **Zero Data Retention:** No databases. No logs. Your prompts and images exist only in your browser's memory and are destroyed upon exit.

### 🛡️ Core Features
* **Uncapped Synthesis:** No corporate guardrails. Speak directly to the neural net.
* **Sovereign Auth:** OAuth integration with `enter.pollinations.ai` to link/unlink your compute credits.
* **Dynamic Canvas:** Native support for 16:9, 9:16, 4:5, and 1:1 aspect ratios.
* **Raw Byte Downloads:** Direct-to-disk asset downloading via Blob URLs to prevent cross-origin resource sharing (CORS) blocks.

### 🚀 Deployment Instructions

Because the architecture relies on pure client-side execution and a basic PHP wrapper, deployment is nearly instantaneous.

1. Clone the repository to your local machine or server.
2. Open `index.php` and locate the `POLLINATIONS_APP_KEY` constant.
3. Inject your public API Key (`pk_...`) generated from [enter.pollinations.ai](https://enter.pollinations.ai).
4. Host the directory on any standard Apache/Nginx web server (PHP 7.4+ recommended).
5. Ensure your domain is added to your Pollinations App settings for OAuth redirection.
